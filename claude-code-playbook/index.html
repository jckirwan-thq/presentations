6. Lifecycle event hooks

"Free your mind."— Morpheus

Hooks fire on session lifecycle events. ~/.claude/settings.json wires command-line tools to each event. Multiple tools per event run in sequence.

Event → what fires

  • SessionStart → caveman-activate · icm hook start · agent-deck
  • UserPromptSubmit → caveman-mode-tracker · icm hook prompt
  • PreToolUse:Bash → rtk proxy (rewrites command)
  • PreCompact → icm hook compact (snapshot)
  • SessionEnd → icm hook end · agent-deck
  • Stop · Notification · PermissionRequest → agent-deck

Why each matters

  • SessionStart — load memory + activate compression before first prompt
  • UserPromptSubmit — inject context (auto-recall ICM, keep caveman alive)
  • PreToolUse:Bash — token-killer transparent rewrite
  • PreCompact — checkpoint long sessions to ICM before context trim
  • SessionEnd — store progress without you having to remember

Full JSON in handouts §8.

7. Plans live in markdown

"Follow the white rabbit."— Trinity

Plans are .md files. Symlink the plans/ folder out of the repo into whatever markdown tool you like. Plans commit with code; the tool gives you the editor.

Sample plan structure

# TRA-23186 · pre-push agent review

## Why
Catch convention violations before
remote review. Run affected tests +
parallel Claude agents.

## Scope
- Husky pre-push hook
- Affected pkg gate (turbo)
- Opt-in via push option
- Severity classifier

## Plan
1. Detect claude CLI + TTY
2. Run turbo affected tests
3. If --o agent-review, dispatch
4. Severity → interactive prompt

## Out of scope
- CI mirror (separate ticket)

## Done when
- 23 test suites pass
- Hook warns gracefully if no CLI
- Manual smoke test on main

Markdown editor options

  • Obsidian (JC's choice) — PARA vault, links, embeds, graph view
  • VS Code — already open, Markdown All in One ext, preview side-by-side
  • iA Writer — distraction-free, paid
  • Typora — WYSIWYG, paid
  • Logseq — outliner + bidirectional links, free
  • Plain text editor — nano, vim, helix

Pick whatever you'll actually open. The symlink doesn't care.

  • Never implement directly — delegate to specialist agent

  • Named Tailwind utilities, never arbitrary [300]
