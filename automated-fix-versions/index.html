A Task With No End

The gods condemned Sisyphus to roll a boulder up a hill — only to watch it roll back down, every time. Effort that never stays done.

That was our Fix Versions

  • Every merge → someone has to remember to set the Fix Version in Jira.
  • People forget. The boulder rolls back.
  • Versions drift from what actually shipped — release notes & reporting can't be trusted.
  • Years of it. The same hill, every day.

What if the boulder rolled itself?

The curse · manual · forgettable · eternal

Before · Push the Boulder, by Hand

Every merge is a manual climb. Miss one and the whole release record is suspect — and you only find out at release time, the worst possible moment.
Before · by hand · every single merge

After · A Machine Rolls the Boulder

GitHub Action jira-fixversion-on-merge.yml. Fires only on a genuine merge. No checkout, permissions: {}, secrets masked, 5-min timeout. The boulder rolls itself — every time, unasked.
After · the machine never forgets
