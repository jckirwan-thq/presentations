The Labour Without End

Sisyphus, condemned to heave the same stone up the same hill — and watch it thunder back down, for all eternity. Toil that never stays finished.

That was our Fix Versions

  • Every merge → a mortal must remember to set the Fix Version in Jira.
  • Mortals forget. The stone rolls back down.
  • Versions drift from what truly shipped — release notes & reporting turn to myth.
  • Years of it. The same hill, every single day.

What if the stone could climb the hill itself?

The curse · manual · forgettable · eternal

Before · The Stone, Heaved by Hand

Every merge is a fresh climb. Miss one and the whole release record is suspect — discovered only at release time, the cruellest hour.
Before · by hand · every single merge

After · A Machine Bears the Stone

GitHub Action jira-fixversion-on-merge.yml. Fires only on a true merge. No checkout, permissions: {}, secrets masked, 5-min timeout. The stone climbs alone — every time, unbidden.
After · the machine never forgets
