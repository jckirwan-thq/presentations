Scaffolding owns every registration

tradify-frontend-platform/scripts/

Generator scripts are the source of truth for handling complexity — generating MFEs, API calls and API schemas. The rule, codified in repo CLAUDE.md: never hand-write a plugin, slice, or endpoint — the generator owns registration in every manifest, env file, package.json and Turbo task. Manual scaffolding fails the pre-commit hook.

GeneratorScaffolds + auto-registers
generate-mfe.tsFull MF plugin + host wiring: plugins-manifest.json, rspack remotes/envMap, root .env, remote.d.ts, App.tsx route/nav — zero hand-edits
generate-evo-mfe.tsWraps a plugin into the Access-Evo host: federation share-scope loader, per-env config — same plugin ships to parent-company shell
generate-codegen-api.tsRTK Query slice from the committed shared-api OpenAPI schema; types stay lockstep with backend
agent-mode.tsThe --agent / --non-interactive convention shared by every generator (see Pillar 2)

Reusable payoff: a new MFE is one command. Future slices inherit correct registration for free — and these generators are now the source of truth Nikita, Jason, Alan and Veronika build against.

Platform primitives + MF singleton contract

  • SHARED_SINGLETONS in generate-evo-mfe.ts — pinned react / redux / frontend-context

tradify-agentic-tools — one home for the team's agentic workflow

tradify-agentic-tools/

Started as tradify-agentic-skills — skills, agents and a session-hooks library I built to share with the team, so anyone could leverage user-level and repo-level skills from one session with full context and minimal token usage. After collaborating with Alan I merged the two libraries into tradify-agentic-tools: one stop home for Tradify's agentic workflows, docs and agents that works in both Claude Code and Devin. A single setup.sh / setup.ps1 symlinks it into every engineer's environment — edits propagate to everyone on the next session.

Skills · agents · session hooks

  • Skills: react-19, typescript, tailwind-v4, module-federation, shadcn, dotnet, entity-framework…
  • Agents: engineering-manager, frontend/backend/angularjs/ux/e2e tech-leads, devops, security-reviewer
  • Session-hooks library — full context, minimal token usage

Distribution mechanism

  • ~/.claude/agents + ~/.claude/skills symlinked (never copied)
  • Aggregates each repo's .claude/agents into tradify/.claude/agents
  • generate-agents-md emits AGENTS.md → Copilot / Cursor parity

Adoption: a talk at the AI meeting got the team excited — Rafael, Wayne and Alan now contribute back. One clone + one script = the whole roster, for every engineer, in Claude or Devin.

Pre-push Claude agent review — into the git workflow

TRA-23186

Runs the repo's specialist agents against the push diff locally, before code leaves the machine.

Two layers

  • Always-on: affected-package test + coverage gate on every git push
  • Opt-in: agent review via git push -o agent-review / git agent-push
  • Agents selected by matching paths: globs to changed files, run in parallel

Graceful for non-adopters

  • No claude CLI? warns and exits 0 — never blocks teammates
  • Findings classed blocking / info
  • Interactive auto-fix / commit-plan / quit

Brings agent review into the standard workflow as a free safety net — alongside /start-ticket, /tra-pr-review and /tra-pr-address-comments, which streamline starting work, reviewing and actioning PR feedback. A human always reviews, resolves and merges. Plan: plans/TRA-23186-claude-agent-pre-push-hook.md.
