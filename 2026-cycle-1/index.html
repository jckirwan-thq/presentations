II · Foundation · what got built
02 / 09

Where we’re going,
we don’t need jQuery.

The legacy web-frontend — born when jQuery was still cool and $scope.$apply() was a love language — stepped politely off the timeline. In its place, tradify-frontend-platform: a Module-Federation host that contains every MFE and lets them speak through one portable API layer, @tradify/frontend-context. Great Scott!

/enquiries — first surface migrated end-to-end. Seven phases of parity with the Angular original; the proof the platform is real. Stands on three sub-pillars:
/me — session, CSRF, identity. Bootstrap today; GET /v2/api/v1/me tomorrow — same contract, zero consumer churn.
/analytics — page-view telemetry, dev dashboard, parity with Angular events (TRA-23263).
AngularJS message bridge — postMessage choreography between the legacy shell and React: bootstrap, route sync, navigation interception, sidebar totals.
/feature-adoption — upsell banners & onboarding panels; the carrot when an entitlement gate denies access.

β ENTITLEMENT GATES · PLAN-TIER PADLOCKS

Doc Brown’s padlock, ported.

AngularJS shipped <upsell-padlock> — a plan-tier gate riveted onto the 2014 dashboard. Re-implementing it in an MFE that boots before bootstrap arrives surfaced two paradoxes at once: a contract the platform didn’t have, and a loading state nobody wanted to see. 

<EntitlementGate
  entitlement="purchaseOrders"
  fallback={<FeatureAdoptionPanel />}
  optimistic
>
  <PurchaseOrderList />
</EntitlementGate>
  • useEntitlement — reads bootstrap.entitlements[key] off the /me slice; resolves granted · denied · loading.
  • optimistic — render children while bootstrap is in flight; swap to fallback if denied. Trades a brief flash for happy-path speed (off for paid-only content).
  • FeatureAdoptionPanel — the carrot inside the padlock: heading, body, primary CTA, dismiss/remind. The upsell when access is denied or the onboarding banner when the feature exists but isn't set up yet.
