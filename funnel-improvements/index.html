The Woodcutter's Mistake

Two woodcutters. One saws flat-out all day. The other stops every hour to sharpen — and fells more timber by dusk.

The crew only had one gear

  • Every job ran the full funnel — orchestrator, Work Order, 5 critics, sign-off.
  • Right for a PR. Wasteful for fast local iteration you re-review at PR time anyway.
  • The blade dragged: tokens up, context filling, compaction churn.

So give the crew a second gear — a thin mode. Thick stays the default; thin trims the local ceremony, not the review.

Habit 7 · self-renewal

Thick · The Full Funnel (default)

~7 agent contexts per job — each critic re-reads the diff and feeds findings back into main. Tokens burn even on a one-line change.
Thick mode · the full funnel (default)

Thin · The Honed Edge

Single-repo = one specialist + Nev's advisory pass: the manifest picks the deepest specialist (context-architect / ui / platform / bundle / devx), and approach-critic-nev rides along as a non-blocking "right shape?" check — on every specialist hop, including each repo's specialist on the cross-repo path. The breakage critics — incl. security unless --thin-security on — defer to the PR gate, where tra-pr-review still runs the full crew.
Thin mode · manifest-first · deferred, not removed
